# บทที่ 1 Overview #

เมื่อก่อนผมเป็นคนหนึ่งที่เกิดข้อสงสัยในใจว่า git คืออะไร และมาช่วยให้งานมันง่ายขึ้นได้อย่างไร เพราะเท่าที่ดูแล้วมันจะต้องเรียนรู้เพิ่มและทำงานเพิ่มมากกว่าเดิม แต่หลังจากที่ผมใช้งาน git จนเข้าใจ ผมพูดได้คำเดียวว่า มันเป็นเครื่องมือที่ทำให้ชีวิตของโปรแกรมเมอร์มีความสุขขึ้น 

สำหรับคนทีไม่เคยใช้โปรแกรมใดๆเข้ามาช่วยจัดการงานทางด้านดูแล sourcecode เวลาจะแบ็กอัพงานทีนึ่ง ก็จะก้อปปี้งานทั้งโฟลเดอร์แล้วอาศัยตั้งชื่อเป็น วัน-เดือน-ปี เอา ของบางคนอาจจะเติมเวลาเข้าไปด้วย การทำอย่างนี้เหมือนจะเป็นประโยชน์ แต่โค้ดที่แบ็กอัพไว้หากจะให้เอากลับมาใช้งานจริงๆ ถามว่าจะมีสักกี่คนกล้าเอามันโยนขึ้นโปรดักชั่นได้โดยไม่ต้องกังวล  ส่วนใหญ่แล้วไม่กล้า ซ้ำร้ายการทำแบ็กอัพบ่อยๆ เป็นการเปลืองฮาร์ดดิสก์ (ถึงดิสก์ปัจจุบันไม่แพงแล้ว แต่ทำไมต้องทำอย่างนี้ล่ะ ในเมื่อเรามีทางเลือกที่ดีกว่า)

สำหรับคนที่ใช้ git มันจะไม่มีปัญหาดังกล่าวข้างต้นเลย 

ผมอยากให้ดูภาพการใช้งานจริงของ git เพื่อเป็นการทำความเข้าใจว่าแท้จริงแล้ว git มันช่วยอะไรเราบ้าง


-เก็บการเปลี่ยนแปลงของโค้ดทุกบรรทัด สีแดงคือลบออก สีเขียวคือเพิ่มเข้ามา

![](images/chapter-1/2014-12-15_1405.png)


-รู้ได้ว่าโค้ดแต่ละบรรทัดใครเป็นคนแก้ มีประโยชน์อย่างมากเมื่อการเพิ่มโค้ดใหม่เข้าไป หรือแก้โค้ดเก่าแล้วเกิดบักขึ้น เมื่อเจอบรรทัดที่ก่อบักและรู้ว่าใครเป็นคนทำ เราสามารถถามเหตุผลการแก้โค้ดบรรทัดนั้นๆได้

![](images/chapter-1/2014-12-15_1409.png)


-โค้ดที่ commit เก็บไว้ใน git แล้วทุกบรรทัดจะไม่หายไปไหน เราจะเรียกคืนโค้ดกลับมาเมื่อไรก็ได้ แม้แต่โค้ดที่เราลบมันไปแล้วก็ตาม

![](images/chapter-1/2014-12-15_1415.png)


-สามารถทดสอบเขียนโค้ด โดยไม่ต้องกลัวโค้ดจะพังด้วยกระบวนการการแตก branch

![](images/chapter-1/2014-12-15_1425.png)


-รู้การเปลี่ยนแปลงของโค้ดทุกๆบรรทัด ทำให้ลดปัญหา "คิดไม่ออกว่าเราทำอะไรไปบ้าง" อีกทั้งสามารถยกเลิกการเปลี่ยนแปลงโค้ดได้อย่างง่ายดายเพียงแค่คลิกปุ่ม Discard ทุกอย่างที่ถูกแก้ไขไปก็กลับมาเหมือนเดิม

![](images/chapter-1/2014-12-15_1429.png)


เหล่านี้เป็นคุณสมบัติเพียงเล็กน้อยเท่านั้น Git ยังมีดีอีกเพียบ
